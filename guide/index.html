💼 Services Flow (the client journey)

This is the core of your business. Every client moves through these 6 steps:

1

Scope

Client visits /services/wizard.html → answers 6 questions → gets a package recommendation + price estimate. Brief saved to their browser.

/services/wizard.html
2

Register

Brief saved to localStorage (no backend). Client can revisit /services/client-dashboard.html anytime.

localStorage: vn_client_briefs
3

Contract

Client clicks "Generate Contract" → /services/contract.html auto-generates an 11-section agreement from the brief → PDF download.

/services/contract.html
4

Pay

Client picks a plan (50/50, milestone, monthly) + method (M-Pesa, Flutterwave, Stripe) → /services/payment.html → emails you to confirm.

/services/payment.html
5

Track

Client watches their brief status change: Draft → Contracted → In Progress → Delivered (in the client dashboard).

brief.status field
6

Launch

You deliver. Client gets training + support. You collect a testimonial → add to homepage.

index.html #testimonials
Action needed: Replace the placeholder M-Pesa Paybill, Flutterwave link, and Stripe link in services/payment.js (the METHODS array) with your actual merchant details.

📝 Content Pipeline

How to publish a new blog article or guide.

1
Create the article file

Copy blog/articles/building-50-agent-dag.html as a template → rename to your slug (e.g. my-new-article.html) → edit content.

2
Add to posts.json

Edit blog/posts.json → add a new entry to the posts array with title, excerpt, date, slug, url, tags, readTime, wordCount.

3
Regenerate RSS feed

Run python3 scripts/build_rss_feed.py → updates /feed.xml automatically.

4
Update sitemap

The GitHub Action update-sitemap.yml runs on push — it auto-adds new articles to sitemap.xml.

5
Push to GitHub

git add -A && git commit -m "new article: ..." && git push → GitHub Pages deploys automatically (gh-pages branch).

Article template: Every article uses /services/guides/guides.css for styling. Use <div class="callout tip|warning|danger|info"> for callouts, <div class="prompt-card"> for before/after prompts, <div class="before-after"> for comparisons.

📊 Analytics

Not yet set up. Follow these steps:
1
Create GA4 property

Go to analytics.google.com → Admin → Create Property → "victorndunda.com" → Web → enter URL → get Measurement ID (format: G-XXXXXXXXXX).

2
Add the GA4 tag

Replace GA_MEASUREMENT_ID in index.html (and all other HTML pages) with your G- ID. The tag is already wired in the <head> of each page (commented out by default).

3
Verify Search Console

Go to search.google.com/search-console → Add property → URL prefix → https://victorndunda.com → verify via HTML tag (add the meta tag to index.html).

4
Submit sitemap

In Search Console → Sitemaps → submit https://victorndunda.com/sitemap.xml.

📧 Email Routing (info@victorndunda.com)

To route info@victorndunda.com → mututandunda@gmail.com, set up DNS MX records with your domain registrar (where you bought victorndunda.com).

This requires DNS access. You can't do this from the code — log into your domain registrar (Namecheap, GoDaddy, Cloudflare, etc.) and add these records:

🔧 Maintenance

📋 Resume PDF

Edit scripts/build_resume_pdf.py → run python3 scripts/build_resume_pdf.py → PDF regenerates.

📡 RSS Feed

Edit blog/posts.json → run python3 scripts/build_rss_feed.py → /feed.xml regenerates.

💾 Service Worker

Bump CACHE_VERSION in sw.js (e.g. v2.7.0 → v2.8.0) when you change cached files. Users get the new version on next visit.

🗂️ Sitemap

Auto-updates via .github/workflows/update-sitemap.yml on every push to main.

🚀 Deploy

Push to main → GitHub Actions builds → deploys to Pages. If Actions fails, push main to gh-pages: git push origin main:gh-pages --force

🔐 Change Password

Edit CONFIG.FALLBACK_AUTH.salt + hash in /auth.js (shared by dashboard and jobs). Generate new hash with: node -e "const c=require('crypto');const s=c.randomBytes(16);const p='YOUR_PASSWORD';c.pbkdf2(p,s,100000,256,'sha256',(e,k)=>console.log('salt:',s.toString('base64'),'hash:',k.toString('base64')))"
